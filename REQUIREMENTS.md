# Business Requirements — react-page-fx

## 1. Назначение

`react-page-fx` — React-библиотека для мобильно-ориентированной стековой навигации между страницами с анимированными переходами. Решает задачу организации экранов в SPA-приложениях с UX, близким к нативным мобильным приложениям: слайд, fade, blur.

**Проблема, которую решает:**
React Router и аналоги строятся вокруг URL-маршрутизации. Для мобильных WebApp, Telegram Mini Apps, PWA и embedded-интерфейсов URL-логика избыточна. Нужен простой стек страниц с плавными переходами — без router, без history API, без конфигурации маршрутов.

---

## 2. Целевая аудитория

- Frontend-разработчики, строящие мобильные WebApp, PWA, Telegram Mini Apps
- Проекты, где URL-маршрутизация не нужна или нежелательна
- Приложения с фиксированным phone-frame layout (390px и подобные)

---

## 3. Ключевые концепции

### 3.1 Стек страниц

Навигация строится как стек (LIFO):
- `nextPage()` — добавляет страницу поверх текущей
- `backPage()` — убирает верхнюю страницу со стека

Одновременно в DOM могут находиться несколько страниц. Анимация перехода выполняется через CSS-переходы, управляемые data-атрибутами.

### 3.2 Жизненный цикл страницы

```
offscreen → entering → active → exiting → (удалена из DOM)
```

- `offscreen` — страница в DOM, но не видна (используется для preload)
- `entering` — запущен CSS-переход входа
- `active` — переход завершён, страница активна
- `exiting` — запущен CSS-переход выхода; по окончании таймера страница удаляется

### 3.3 Эффекты

Эффект задаётся отдельно для открываемой страницы (`effect`) и для родительской страницы в момент, когда дочерняя открыта поверх неё (`parentEffect`).

## 4. Функциональные требования

### 4.1 Инициализация

- Библиотека предоставляет один корневой компонент, который оборачивает всё приложение
- Первая страница передаётся при монтировании
- Компонент должен монтироваться ровно один раз; React StrictMode не должен приводить к дублированию начальной страницы

### 4.2 Навигация вперёд (`nextPage`)

- Принимает React-компонент и опции (эффект, props, имя, back button, parentEffect, onBack)
- Новая страница добавляется поверх текущей
- Анимация запускается через двойной `requestAnimationFrame` после первого paint, чтобы CSS-transition корректно отработал
- После завершения анимации страница переходит в состояние `active`

Обязательные поля:
- `Component` — React-компонент страницы

Необязательные поля:
- `pageName` — строковый идентификатор (используется в `deletePages`)
- `effect` — эффект входа (`none` | `slide` | `fade` | `blur`), по умолчанию `none`
- `props` — произвольные props, передаваемые в компонент
- `showBackButton` — показывать ли кнопку «назад», по умолчанию `false`
- `onBack` — callback, вызываемый в момент начала анимации выхода
- `parentEffect` — эффект, применяемый к родительской странице, пока эта страница открыта (`none` | `slide` | `blur`)
- `className` — дополнительный CSS-класс на обёртке страницы

### 4.3 Навигация назад (`backPage`)

- Убирает текущую страницу со стека
- Вызывает `onBack`-callback до начала анимации
- Одновременно запускает exit-анимацию страницы и снимает `parentEffect` с родительской страницы
- По истечении `duration[effect]` страница удаляется из DOM

### 4.4 Preload

- `preloadPage(options)` монтирует страницу в DOM в состоянии `offscreen` (не видима, не интерактивна)
- Возвращает `PreloadHandle` с полями:
  - `id` — уникальный идентификатор
  - `updateProps(props)` — обновляет props смонтированной страницы без размонтирования
- Передача `PreloadHandle` в `nextPage()` активирует страницу мгновенно (без задержки на рендер)
- Монтирование происходит в следующем tick через `setTimeout(0)`, чтобы не конфликтовать с текущим render-циклом

**Сценарий использования:** загрузить данные и отрендерить страницу заранее → показать мгновенно по действию пользователя.

### 4.5 Удаление страниц (`deletePages`)

- Принимает массив `pageName`
- Удаляет все страницы с указанными именами из стека без анимации
- Используется для очистки кешированных страниц, которые уже не нужны

### 4.6 Эффекты переходов

| Эффект  | Описание                                                  |
|---------|-----------------------------------------------------------|
| `none`  | Мгновенный переход без анимации                           |
| `slide` | Страница въезжает справа; при выходе уезжает вправо       |
| `blur`  | Родительская страница размывается, пока дочерняя открыта  |
| `fade`  | Страница появляется/исчезает через opacity                |

Эффект `blur` применяется **только как `parentEffect`** — он воздействует на страницу позади, а не на открываемую.

Длительности эффектов задаются в миллисекундах и могут быть переопределены через конфиг.

### 4.7 Back Button

- По умолчанию библиотека рендерит встроенную кнопку «назад», если `showBackButton: true`
- Кнопка позиционируется через CSS-класс `{baseClass}__back-button`
- Через `config.BackButton` можно передать кастомный компонент с интерфейсом `{ onClick: () => void }`
- Кастомный компонент полностью заменяет встроенный во всех страницах, где `showBackButton: true`

### 4.8 Конфигурация (`config`)

Опциональный объект, передаваемый в корневой компонент:

| Поле         | Тип                                      | По умолчанию  | Описание                                        |
|--------------|------------------------------------------|---------------|-------------------------------------------------|
| `durations`  | `{ fade?, slide?, blur? }` (мс)          | 300 / 300 / 150 | Длительности анимаций по эффектам              |
| `baseClass`  | `string`                                 | `pfx-page`    | CSS-класс обёртки страниц                       |
| `BackButton` | `ComponentType<{ onClick: () => void }>` | встроенный    | Кастомный компонент кнопки «назад»              |

### 4.9 Навигационный хук (`usePageNavigator`)

- Доступен внутри любого компонента в дереве
- Возвращает объект `PageNavigator`, привязанный к текущей странице
- Вызов вне дерева корневого компонента — ошибка с понятным сообщением
- `nextPage` и `preloadPage` автоматически устанавливают текущую страницу как родителя

## 5. Нефункциональные требования

### 5.1 Производительность

- Анимации реализуются исключительно через CSS-переходы (`transition`) — без JS-анимационных циклов
- Стейт обновляется минимально: только затронутые записи в стеке
- Callbacks (`push`, `pop`, `preload`, `deleteByNames`, `advanceStep`) стабильны между рендерами (`useCallback` без лишних зависимостей) — не провоцируют ре-рендер дочерних компонентов
- Двойной `requestAnimationFrame` гарантирует корректный paint до старта CSS-transition без перформанс-оверхеда

### 5.2 Совместимость

- React >= 18 (peer dependency)
- TypeScript — полная типизация публичного API, без `any` в пользовательском коде
- Dual package: ESM + CJS, `.d.ts` + `.d.cts`
- Не зависит от React Router, history API, URL или window.location

### 5.3 Размер

- Минимальный bundle footprint: CSS инжектируется инлайн через `<style>` тег без внешних зависимостей
- Zero runtime dependencies (только peerDeps: react, react-dom)

### 5.4 Тестируемость

- Переходы управляются data-атрибутами (`data-step`, `data-effect`, `data-child-effect`) — удобны для assertions в тестах
- Публичный API полностью покрывается интеграционными тестами через `@testing-library/react`
- Fake timers совместимы с `setTimeout`-логикой удаления страниц

### 5.5 Изоляция стилей

- Все CSS-правила библиотеки скопированы под `baseClass`-префикс
- Стили не вытекают за пределы компонентов библиотеки
- Потребитель может переопределить `baseClass` для изоляции от других экземпляров

## 6. Граничные случаи и ограничения

| Ситуация | Ожидаемое поведение |
|----------|---------------------|
| `backPage()` на единственной странице | Ничего не происходит (`pop` проверяет наличие записи) |
| `nextPage(handle)` с несуществующим preload ID | Ничего не происходит (silent no-op) |
| `deletePages` с именами, которых нет в стеке | Ничего не происходит |
| `usePageNavigator()` вне дерева | Бросает ошибку с текстом `usePageNavigator must be called inside a <PageFX /> tree` |
| React StrictMode | Двойной вызов эффектов защищён `useRef`-флагом; дублей начальной страницы не возникает |
| Несколько корневых компонентов на странице | Каждый имеет независимый стек и контекст |

---

## 7. Открытые вопросы (к обсуждению)

- **Имя корневого компонента** — `PageFX` звучит нечитаемо; рассматриваются варианты `PageStack`, `ScreenStack` и другие
- **Children API** — передача начального экрана через `children` вместо `initPage` prop (`<PageStack><HomeScreen /></PageStack>`)
- **`durations`** → `transitionDurations` или `animationDurations` — сделать назначение поля очевиднее
- **`baseClass`** → `pageBaseClassName` — привести в соответствие с React-конвенцией именования className-пропов
- **BackButton в демо** — добавить пример использования кастомного `BackButton` через `config`
